## 1. Class Hierarchy and Core Components

### Module Definition DSL Architecture

```
ModuleDefinitionBuilder (Main DSL builder)
├── Module (abstract base class)
│   ├── _appContextHolder: WeakReference<AppContext>
│   ├── runtime: Runtime
│   ├── registry: ModuleRegistry
│   └── sendEvent(): void
├── InternalModuleDefinitionBuilder (extends ObjectDefinitionBuilder)
│   ├── name: String?
│   ├── viewManagerDefinitions: Map<String, ViewManagerDefinition>
│   ├── eventListeners: Map<EventName, EventListener>
│   └── classData: List<ClassDefinitionData>
└── ObjectDefinitionBuilder (Base for reusable objects)
    ├── syncFunctions: Map<String, SyncFunctionComponent>
    ├── asyncFunctions: Map<String, AsyncFunctionComponent>
    ├── properties: Map<String, PropertyComponent>
    ├── constants: Map<String, ConstantComponent>
    └── eventsDefinition: EventsDefinition?

ModuleDefinitionData (Final compiled definition)
├── name: String
├── objectDefinition: ObjectDefinitionData
├── viewManagerDefinitions: Map<String, ViewManagerDefinition>
├── eventListeners: Map<EventName, EventListener>
├── registerContracts: Callable?
└── classData: List<ClassDefinitionData>

ObjectDefinitionData
├── legacyConstantsProvider: () -> Map<String, Any?>
├── syncFunctions: Map<String, SyncFunctionComponent>
├── asyncFunctions: Map<String, BaseAsyncFunctionComponent>
├── eventsDefinition: EventsDefinition?
├── properties: Map<String, PropertyComponent>
├── constants: Map<String, ConstantComponent>
└── functions: Iterator (concat of sync + async)
```

## 2. Complete DSL Components List

### Module-Level Components (ModuleDefinitionBuilder)

| Component | Signature | Purpose |
|-----------|-----------|---------|
| **Name** | `Name(name: String)` | Sets module name exposed to JavaScript |
| **Function** | `Function(name, body: (args) -> R)` | Synchronous function (8 overloads, 0-7 params) |
| **AsyncFunction** | `AsyncFunction(name, body: (args) -> R)` | Async function returning value (8 overloads) |
| **AsyncFunction** (Promise) | `AsyncFunction(name, body: (args, Promise) -> R)` | Async function with explicit Promise (8 overloads) |
| **SuspendBody** | `SuspendBody(block: suspend (args) -> R)` | Kotlin suspend coroutine (8 overloads) |
| **Events** | `Events(vararg events: String)` | Declares event names module can emit |
| **Events** (Enum) | `Events<T : Enumerable>()` | Event names from enum |
| **OnStartObserving** | `OnStartObserving(eventName, body)` | Lifecycle: specific event listener added |
| **OnStartObserving** (All) | `OnStartObserving(body)` | Any event listener added |
| **OnStopObserving** | `OnStopObserving(eventName, body)` | Lifecycle: specific event listeners removed |
| **OnStopObserving** (All) | `OnStopObserving(body)` | All listeners removed |
| **Property** | `Property(name).get { }.set { }` | Read/write property |
| **Constant** | `Constant(name, body: () -> T)` | Read-only constant |
| **OnCreate** | `OnCreate(body)` | Module initialization hook |
| **OnDestroy** | `OnDestroy(body)` | Module deallocation hook |
| **View** | `View(viewClass, body: ViewDefinitionBuilder)` | View manager definition |
| **Class** | `Class(name, body: ClassComponentBuilder)` | Class/SharedObject definition |
| **RegisterActivityContracts** | `RegisterActivityContracts(block)` | Activity result registration (Android-only) |
| **OnActivityEntersForeground** | `OnActivityEntersForeground(body)` | Activity resume (Android-only) |
| **OnActivityEntersBackground** | `OnActivityEntersBackground(body)` | Activity pause (Android-only) |
| **OnActivityDestroys** | `OnActivityDestroys(body)` | Activity destruction (Android-only) |

### View Components (ViewDefinitionBuilder<T>)

| Component | Signature | Purpose |
|-----------|-----------|---------|
| **Name** | `Name(viewName)` | View name in JS |
| **Prop** | `Prop(name, setter: (view, value) -> Unit)` | View property setter (8 overloads) |
| **OnViewDestroys** | `OnViewDestroys(body: (view) -> Unit)` | View cleanup |
| **OnViewDidUpdateProps** | `OnViewDidUpdateProps(body: (view) -> Unit)` | After props batch applied |
| **AsyncFunction** | Same as module-level | Methods callable from JS on view |

### Class Components (ClassComponentBuilder<SharedObjectType>)

| Component | Signature | Purpose |
|-----------|-----------|---------|
| **Constructor** | `Constructor(body: () -> SharedObjectType)` | Instance creation (required) |
| **Function** | With `self: SharedObjectType` param | Instance methods |
| **AsyncFunction** | With `self: SharedObjectType` param | Async instance methods |
| **Property** | getter/setter receive `self` | Instance properties |
| **Events** | Same as module | Instance events |
| **UseTrait** | `UseTrait(trait)` | Mixin behavior |

## 3. Function Component Architecture

### Hierarchy

```
AnyFunction (abstract base)
├── name: String
├── desiredArgsTypes: Array<AnyType>
├── convertArgs(args, appContext): Array<Any?>
├── takesOwner: Boolean
│
├── SyncFunctionComponent
│   └── callUserImplementation(args, appContext): Any?
│       Input: JS arguments → Convert → Call body → Convert result → Output
│
├── BaseAsyncFunctionComponent (abstract)
│   ├── queue: FunctionQueue (DEFAULT, MAIN, or CustomQueue)
│   └── runOnQueue(queue): BaseAsyncFunctionComponent
│
├── AsyncFunctionComponent (value return, auto-Promise)
│   └── callUserImplementation(args, promise, appContext): void
│
├── AsyncFunctionWithPromiseComponent (explicit Promise param)
│   └── Same as AsyncFunctionComponent
│
└── SuspendFunctionComponent (Kotlin suspend / C# async)
    └── Coroutine-based execution → auto-Promise
```

### Function Queue Dispatch

```
Queues.DEFAULT  → Background thread pool (modulesQueue coroutine)
Queues.MAIN     → UI thread (mainQueue)
CustomQueue     → Custom CoroutineScope / TaskScheduler
```

## 4. Type Conversion System

### Supported Types (Comprehensive)

**Primitives:** `Int`, `Float`, `Double`, `Long`, `Boolean`, `String`, `Unit/void`, `Any/object`

**Collections:** `List<T>`, `Map<String, T>`, `Set<T>`, `Pair<A, B>`, `Array<T>`, `IntArray`, `FloatArray`, `DoubleArray`, `BooleanArray`, `ByteArray`

**JS Interop:** `JavaScriptValue`, `JavaScriptObject`, `JavaScriptFunction`, `JavaScriptArrayBuffer`, `TypedArray` (all variants)

**I/O:** `File`, `Path`, `URL`, `URI`

**Utilities:** `Duration` (→ Double seconds), `Record` (custom objects), `Enumerable` (enums), `Either<T, P>` / `EitherOfThree` / `EitherOfFour` (union types), `ValueOrUndefined<T>`, `Promise`

### Type Converter Hierarchy

```
TypeConverter<Type>
├── convert(value, context, forceConversion): Type?
├── getCppRequiredTypes(): ExpectedType
└── isTrivial(): Boolean

NonNullableTypeConverter<Type : Any>
│   └── convertNonNullable(value, context, forceConversion): Type

DynamicAwareTypeConverters<T>
    ├── convertFromDynamic(Dynamic, context, forceConversion): T
    └── convertFromAny(Any, context, forceConversion): T
```

### Key Converters

| Type | Converter | Notes |
|------|-----------|-------|
| Primitives | `createTrivialTypeConverter()` | Direct cast, no conversion |
| `List<T>` | `ListTypeConverter` | Recursive element conversion |
| `Map<String, T>` | `MapTypeConverter` | Key/value conversion |
| `Record` | `RecordTypeConverter` | Reflection-based field mapping |
| `Enum<T>` | `EnumTypeConverter` | String → enum by name |
| `Either<A, B>` | `EitherTypeConverter` | Deferred conversion on access |
| `TypedArray` | `TypedArrayTypeConverter` | JSI ArrayBuffer access |
| `Date` | `DateTypeConverter` | Milliseconds ↔ Date |
| `Duration` | `DurationTypeConverter` | Double seconds ↔ Duration |

### Conversion Flow (JS → Native → JS)

```
JavaScript Value
  ↓ JSI Layer (C++)
Dynamic/JavaScriptValue
  ↓ TypeConverter.convert() [Kotlin/Java]
  ├─ Check if trivial → return as-is
  ├─ Obtain correct converter for target type
  └─ Apply conversion logic
  ↓
Native Type (Kotlin Object)
  ↓ Function Execution
Native Return Type
  ↓ ReturnType.convertToJS()
  ├─ null/Unit → null
  ├─ Primitive → direct
  ├─ Record → JS object
  ├─ Collection → JS array
  ├─ Enum → string
  └─ Long/Duration → double
  ↓
JavaScript Value
```

## 5. Records and Custom Types

### Record Interface and Annotations

```kotlin
interface Record  // Marker interface

@Field(key: String = "")     // Custom JSON key name
@Required                     // Field is mandatory

// Example:
data class UserRecord : Record {
  @Field("user_id") @Required
  val id: String
  val email: String?           // Optional (nullable)
  @Field("display_name")
  val name: String = "Unknown" // Default value
}
```

### Record Conversion

```
RecordTypeConverter:
  1. Get readable map from JS value
  2. For each property:
     a. Check @Field annotation for key name
     b. Recursively convert field value
     c. Throw FieldRequiredException if @Required but missing
     d. Use default value if present
  3. Construct record via reflection
```

## 6. Module Registry and Lifecycle

### ModuleRegistry Structure

```
ModuleRegistry
├── registry: Map<String, ModuleHolder<*>>
├── eventQueue: List<PostponedEvent>  (startup buffering)
├── isReadyForPostingEvents: Boolean
│
├── register(module, name?)
│   ├─ Create ModuleHolder
│   ├─ Setup coroutine scope
│   └─ Add to registry
│
├── postOnCreate()
│   ├─ Post MODULE_CREATE to all modules
│   ├─ registerActivityContracts()
│   ├─ readyForPostingEvents()
│   └─ flushTheEventQueue()
│
└── post(eventName, sender?, payload?)
```

### ModuleHolder

```
ModuleHolder<T : Module>
├── module: T
├── definition: ModuleDefinitionData
├── name: String
├── jsObject: JavaScriptModuleObject (lazy)
│   └─ Decorator: JSDecoratorsBridgingObject
│       ├─ registerConstants(map)
│       ├─ registerSyncFunction(name, ...)
│       └─ registerAsyncFunction(name, ...)
```

### Lifecycle Order

1. **Module Construction** → `Module()` constructor
2. **AppContext Init** → ModuleRegistry created, modules registered
3. **postOnCreate()** → `MODULE_CREATE` posted, activity contracts registered, events flushed
4. **Runtime** → Functions callable, events emittable
5. **Destruction** → `OnDestroy` event, context cleanup

## 7. Event System

### EventEmitter Architecture

```
EventEmitter (interface)
├── emit(eventName, eventBody: WritableMap?)
├── emit(eventName, eventBody: Map?)
├── emit(eventName, eventBody: Record?)
├── emit(viewId, eventName, eventBody, coalescingKey)

Module.sendEvent() Flow:
  1. module.sendEvent("eventName", bundle)
  2. EventEmitter via appContext.eventEmitter(module)
  3. KModuleEventEmitterWrapper emits to JS
  4. JavaScript listener receives payload
```

### Event Observation Lifecycle

```
OnStartObserving:
  1. JS: emitter.addListener("eventName", callback)
  2. Native: startObserving("eventName") called
  3. Native: OnStartObserving hook executes → allocate resources

OnStopObserving:
  1. JS: subscription.remove()
  2. Native: stopObserving("eventName") (when last listener removed)
  3. Native: OnStopObserving hook executes → release resources
```

## 10. Data Flow Diagrams

### Synchronous Function Call

```
JS: NativeModules.MyModule.add(2, 3)
  ↓ JSI Bridge
C++ HostObject: get("add") → creates JSI function
  ↓ JNI Callback
Kotlin SyncFunctionComponent:
  1. convertArgs([2, 3]) → [2 as Int, 3 as Int]
  2. body([2, 3]) → 2 + 3 = 5
  3. returnType.convertToJS(5) → 5
  ↓ C++ → JS
JS: result = 5
```

### Asynchronous Function Call

```
JS: NativeModules.MyModule.fetchData("id").then(...)
  ↓ JSI Async Handler
Kotlin AsyncFunctionComponent:
  1. Create PromiseImpl
  2. Schedule on queue (DEFAULT=bg, MAIN=UI, Custom)
  3. callUserImplementation(["id"], promise)
  4. User code: promise.resolve(result) or promise.reject(...)
  ↓ Promise settlement
JS: .then(data) or .catch(err)
```

### Event Emission

```
Native: sendEvent("onData", { value: 42 })
  ↓ EventEmitter
  ↓ C++ JSI bridge
JS: emitter.listeners("onData") called with { value: 42 }
```

## 11. C# Adaptation Recommendations

### Language Feature Mapping

| Kotlin | C# | Notes |
|--------|-----|-------|
| `inline fun <reified T>` | `typeof(T)` / generics | C# generics are runtime, not reified |
| Trailing lambda `{ }` | `() => { }` | Builder pattern works similarly |
| `data class` | `record` (C# 9+) | Closest match |
| `suspend` / coroutines | `async/await` / `Task` | C# async is more standard |
| `Type?` nullable | `Type?` (C# 8+) | Direct equivalent |
| Sealed classes | Abstract + sealed hierarchy | Or discriminated unions |
| Extension functions | Extension methods | Direct equivalent |
| `crossinline` | Closure capture | Automatic in C# |

### Architecture Mapping

| Kotlin Component | C# Equivalent |
|-----------------|---------------|
| `ModuleDefinitionBuilder` | `ModuleDefinitionBuilder<TModule>` (generics) |
| Module DSL (trailing lambda) | Fluent builder with method chaining |
| `TypeConverter<T>` | `ITypeConverter<T>` interface |
| `Promise` | `TaskCompletionSource<T>` or custom |
| `EventEmitter` | Custom `IEventEmitter` |
| `AppContext` | `ModuleContext` / `AppContext` |
| Kotlin coroutines | `Task<T>` / `ValueTask<T>` |

### DSL Syntax: Kotlin vs C#

```kotlin
// Kotlin: trailing lambda with implicit receiver
ModuleDefinition {
  Name("MyModule")
  Function("add") { a: Int, b: Int -> a + b }
}
```

```csharp
// C# Option A: Collection initializer
public override ModuleDefinition Definition() => new()
{
    Name("MyModule"),
    Function("add", (int a, int b) => a + b),
};

// C# Option B: Fluent builder
public override ModuleDefinition Definition() =>
    new ModuleDefinitionBuilder()
        .Name("MyModule")
        .Function("add", (int a, int b) => a + b)
        .Build();
```

### Async Handling

```csharp
// Kotlin suspend → C# async/await (natural mapping)
AsyncFunction("fetch", async (string url) =>
{
    var result = await HttpClient.GetAsync(url);
    return result.Content;
});

// Kotlin Promise param → C# TaskCompletionSource
AsyncFunction("fetch", async (string url, TaskCompletionSource<object> tcs) =>
{
    try {
        var result = await FetchAsync(url);
        tcs.SetResult(result);
    } catch (Exception e) {
        tcs.SetException(e);
    }
});
```

### Records

```csharp
// Kotlin Record interface → C# record + attribute
[ExpoRecord]
public record UserInfo(
    [Field("user_id")] [Required] string Id,
    string? Email,
    [Field("display_name")] string Name = "Unknown"
);
```
