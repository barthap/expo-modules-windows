# Prompt: Build Integration — MSBuild, Solution, Example Module

**Agent type:** Plan (then implement)
**Priority:** 3 — requires C# core (#1) and C++ host (#2)
**Dependencies:** Steps 1 + 2

---

```
Set up the build integration so that C# modules compile and deploy alongside
the C++ React Native Windows app with a single F5 in Visual Studio.

## Context

We have:
1. C# core library: dotnet/Expo.Modules.Core/ (Module base class, DSL, interop)
2. C++ host: windows/ExpoModulesWindowsCore/ (TurboModule, JSI HostObject, HostFXR)
3. Example RNW app: example/windows/ExpoModulesWindowsCoreExample/

The C++ app needs to:
- Load dotnet/Expo.Modules.Core.dll at runtime via HostFXR
- Find module DLLs and their .runtimeconfig.json files
- Work in both Debug and Release configurations, x64 and ARM64

## Repository

D:\dev\expo-modules-windows-core
- windows/ExpoModulesWindowsCore/ — C++ .vcxproj
- windows/ExpoModulesWindowsCore.sln — library solution
- dotnet/Expo.Modules.Core/ — C# .csproj
- example/windows/ExpoModulesWindowsCoreExample/ — example app .vcxproj
- example/windows/ExpoModulesWindowsCoreExample.sln — example solution

## What to implement

### 1. Solution structure
- Add C# project to both .sln files (library + example)
- Set build dependencies: C# builds before C++ projects
- Handle mixed C++/C# solution (VS supports this natively)

### 2. MSBuild targets for DLL deployment
Create `windows/ExpoModulesWindowsCore/Expo.Modules.targets`:
```xml
<!-- Included by consuming apps to deploy C# modules -->
<Target Name="BuildExpoModules" BeforeTargets="PrepareForBuild">
  <Exec Command="dotnet build &quot;$(ExpoModulesCorePath)&quot; -c $(Configuration)" />
</Target>

<Target Name="CopyExpoModules" AfterTargets="Build">
  <!-- Copy DLL, PDB, runtimeconfig.json, deps.json to app output -->
  <Copy SourceFiles="..." DestinationFolder="$(OutDir)" />
</Target>
```

Must handle:
- Debug vs Release configuration mapping
- x64 vs ARM64 platform
- All transitive .NET dependencies
- The .runtimeconfig.json file (required by HostFXR)

### 3. Example C# module
Create `example/modules/ExampleModule.cs`:
```csharp
public class ExampleModule : Module
{
    public override ModuleDefinition Definition() => new()
    {
        Name("Example"),
        Function("multiply", (double a, double b) => a * b),
        Function("greet", (string name) => $"Hello, {name}!"),
        AsyncFunction("delayedSquare", async (int n) => {
            await Task.Delay(100);
            return n * n;
        }),
        Constants(new { version = "0.1.0" }),
    };
}
```

### 4. Autolinking-generated module list
For now (before autolinking fork), create a static file:
`example/windows/ExpoModulesProvider.g.cs`
```csharp
// Will be auto-generated by expo-modules-autolinking in the future
public static class ExpoModulesProvider
{
    public static Type[] GetModules() => new[]
    {
        typeof(ExampleModule),
    };
}
```

### 5. Example app wiring
Modify example app to:
- Reference the C# core library
- Include the ExpoModulesProvider
- Pass module type list to C++ host during initialization

### 6. JS-side example
Update example/src/App.tsx to call the C# module:
```tsx
const result = global.ExpoModules.Example.multiply(3, 7);
const greeting = global.ExpoModules.Example.greet("World");
```

### 7. NuGet configuration
- Ensure NuGet.config supports .NET packages
- Add any required NuGet package references to the C# project

### 8. npm package structure
Update package.json `files` array to include:
- dotnet/ directory
- The .targets file
- Any build scripts needed

## Constraints
- Must work with Visual Studio 2022
- F5 builds everything (C# + C++ + deploys + runs)
- Debug symbols work across C# and C++ (mixed debugging)
- No manual build steps required

## Output
Provide all modified files: .sln changes, .targets files, .csproj references,
package.json updates, and the example module code.
