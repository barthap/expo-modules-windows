## What Was Built

A standalone proof-of-concept that validates C++ can load the .NET runtime via HostFXR and call C# methods with `[UnmanagedCallersOnly]` exports. No React Native dependency.

### Test Steps in main.cpp

1. Load HostFXR via `get_hostfxr_path` + `LoadLibraryW`
2. Initialize .NET runtime via `hostfxr_initialize_for_runtime_config`
3. Resolve all 5 C# exports using `load_assembly_and_get_function_pointer` with `UNMANAGEDCALLERSONLY_METHOD`
4. Test `ExpoTest_Add(3, 7)` — expects 10
5. Test `ExpoTest_Multiply(3.5, 2.0)` — expects 7.0
6. Test `ExpoTest_GetModuleInfo` — expects JSON with module name and function list
7. Test `ExpoTest_InvokeFunction` — three sub-tests: add via JSON, multiply via JSON, unknown function error
8. Memory leak test — 1000 invoke+free cycles
9. Per-call overhead benchmark — 10,000 direct `Add` calls with timing

## Key Design Decisions

### Memory Ownership Protocol
- `InvokeFunction` allocates result buffers with `Marshal.AllocHGlobal` on C# side
- C++ reads the data, then calls `FreeBuffer` → `Marshal.FreeHGlobal`
- Same pattern the real RNW host will use

### UNMANAGEDCALLERSONLY_METHOD Sentinel
- Tells HostFXR to look for `[UnmanagedCallersOnly]` attribute
- Correct approach for direct native-to-managed calls
- **Note:** HostFXR resolves by method name on the type, not by the `EntryPoint` string. The `EntryPoint` property is only used for NativeAOT export naming.

### Assembly Discovery
- C++ app searches multiple paths: next to exe, `managed/` subdirectory, relative to CWD
- Works in both development (repo root) and deployment scenarios
